Mdecycu | 週二 05 七月 2022 | Updated on 週一 11 七月 2022

上一次採用 TCExam, 是在 2013 年 11 月 1 日, 多年之後, 這套線上考試系統, 仍然可以使用嗎?

Ubuntu 22.04

為了要將 TCExam 安裝在 Ubuntu 22.04, 首先必須要先處理 /etc/netplan/net.yaml 中的設定:

network:
  ethernets:
    enp0s3:
      dhcp6: false 
      dhcp4: false
      addresses:
        - 140.130.xx.xxx/24
        - 2001:288:6004:xx::xxx/64
      nameservers:
        addresses:
          - 168.95.1.1
          - 2001:b000:168::1
      routes:
        - to: default 
          via: 140.130.xx.254
        - to: ::/0      
          via: 2001:288:6004:xx::254
  version: 2

網路設定完成後, 隨即進行更新與升級:

sudo apt update
sudo apt upgrade
sudo apt autoremove

接下來希望以最小安裝放入桌面, 可以執行

sudo apt install --no-install-recommends ubuntu-desktop

但是因為這個 bug 尚未處理, 因此暫時先建立 /etc/apt/apt.conf.d/phased 應急. 之後應該要將這個檔案刪除, 採用官方除錯後的版本.

接下來要安裝 nginx, 並且讓 php 解譯環境可以與 nginx 合用. 最後則是安裝 Postgresql, 以及 TCExam.

Windows

假如要將 TCExam 安裝在 Windows 10, 則可以安裝 Nginx 1.20.1 and PHP 8.1.7 (vc_2019_for_php8.1.7.7z).

讓 Nginx 可以與 PHP 合用則採下列設定, 表示要由 Nginx 接受外部 request PHP 程式執行, 實際上則使用 fastcgi 在近端 port 9123 執行遠端用戶所要求的 php 程式, 並將結果由 Nginx 代理傳回.

server { 
    listen    98 ssl;

    root c:/tmp/TCExam;

    server_name    your_domain.name;
    ssl_certificate    C:/pj2022/stunnel/config/fullchain.pem;
    ssl_certificate_key    C:/pj2022/stunnel/config/privkey.pem;
    ssl_protocols    TLSv1 TLSv1.1 TLSv1.2;
    ssl_ciphers    HIGH:!aNULL:!MD5;

    location /{ 
        index index.html index.php;
    }

    location ~ \.php$ {
        fastcgi_pass   127.0.0.1:9123;
        fastcgi_index  index.php;
        fastcgi_param  SCRIPT_FILENAME  $document_root$fastcgi_script_name;
        include        fastcgi_params;
    }
}

因此上述流程必須同時啟動 Nginx 與 php-cgi.exe 等兩則伺服器. Nginx 根據上述設定, 在 port 98 啟動, 而 php-cgi.exe 則由下列 nssm service 啟動.

nssm install php8.1.7

Application path: C:\PHP8.1.7\php-cgi.exe
Startup directory: C:\PHP8.1.7
Arguments: -b 127.0.0.1:9123

啟動 php-cgi:

C:\PHP8.1.7\php-cgi.exe -b 127.0.0.1:9123
